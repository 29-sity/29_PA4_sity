Unit Testing
■ Unit testing is a software development process in which the smallest testable parts of an application, called units, are individually scrutinized for proper operation.
■ Unit Testing is conducted by Developers.
■ It is type of whitebox testing

Integration Testing
■ Integration testing is the process of testing the interface between two software units or modules
■ Integration Testing is conducted by Developers.
■ It is type of blackbox testing.

System Testing
It is checking how components interact with each other and the system, was well as testing every possible input/output.
Conducted by a testing team seperate from development team.
Black Box

Acceptance Testing
It is a quality assurance process that ensures the application meets end users approval.
Conducted by customers or users of the software.
Black Box
